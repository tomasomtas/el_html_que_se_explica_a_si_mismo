Introducción

HTML (HyperText Markup Language) es el lenguaje estándar utilizado para estructurar el contenido de las páginas web. Fue desarrollado para permitir la conexión entre documentos mediante enlaces (hipervínculos), lo que dio lugar a la creación de la World Wide Web tal como la conocemos hoy en día. Su principal función es organizar y presentar información de manera estructurada, utilizando etiquetas que determinan la semántica y jerarquía del contenido.

HTML por sí solo no es un lenguaje de programación; es un lenguaje de marcado. Esto significa que su propósito no es ejecutar lógica o algoritmos, sino definir la estructura de un documento. Por ejemplo, con HTML puedes indicar qué partes de un texto son títulos, párrafos, listas, tablas, imágenes o enlaces.

Este lenguaje es la base del desarrollo web frontend, que es la parte visible e interactiva de un sitio web. Sin embargo, HTML no trabaja de forma aislada. Suele estar estrechamente integrado con otras tecnologías clave:

  • CSS (Cascading Style Sheets): Utilizado para dar estilo y diseño a las páginas web. Mientras que HTML define el contenido y su estructura, CSS se encarga de cómo se ve, controlando colores, fuentes, márgenes, posicionamiento, animaciones y más.
  • JavaScript: Lenguaje de programación que añade interactividad y dinamismo a las páginas. Por ejemplo, permite crear menús desplegables, formularios interactivos, efectos visuales y muchas otras funcionalidades avanzadas.

Además de CSS y JavaScript, HTML también se asocia con tecnologías y conceptos modernos que amplían sus capacidades:

  • Frameworks de frontend: Librerías como React, Angular o Vue.js permiten crear interfaces de usuario complejas basadas en componentes, combinando HTML, CSS y JavaScript.
  • APIs web: HTML puede interactuar con APIs para integrar contenido dinámico, como mapas, datos en tiempo real o notificaciones.
  • Backend y bases de datos: Aunque HTML está en el lado del cliente (frontend), se combina con tecnologías backend como PHP, Python, Node.js y bases de datos (MySQL, MongoDB) para generar contenido dinámico y personalizado.

HTML ha evolucionado significativamente desde su creación. Actualmente, la versión más reciente es HTML5, que introdujo nuevas etiquetas semánticas como <article>, <section> y <aside>, además de soporte nativo para audio, video y gráficos vectoriales mediante <audio>, <video> y <canvas>.

En resumen, HTML es la piedra angular del desarrollo web. Define la estructura básica de los sitios y sirve como base para integrar estilos, interactividad y datos dinámicos, permitiendo la creación de experiencias web modernas, accesibles y atractivas.

Estructura básica de HTML

Un documento HTML comienza con la declaración <!DOCTYPE html>, que indica al navegador que el documento está utilizando HTML5, la versión más moderna del lenguaje. A continuación, se estructura mediante las etiquetas principales <html>, <head> y <body>.

Para crear y editar código HTML, necesitas un editor de texto. Algunos de los editores más populares para programadores son:

  • Visual Studio Code (VSC): Un editor gratuito y muy popular que incluye herramientas útiles como resaltado de sintaxis, sugerencias de código y extensiones para trabajar con HTML, CSS, JavaScript y más.
  • Sublime Text: Un editor ligero y rápido, ideal para escribir código HTML y otros lenguajes. Ofrece resaltado de sintaxis y múltiples herramientas de edición.
  • Otros editores: También puedes usar herramientas como Notepad++, Atom o incluso editores en línea como CodePen o JSFiddle para probar y compartir tu código.

Los archivos HTML tienen la extensión .html. Una vez creado el archivo, puedes abrirlo en cualquier navegador web para visualizar cómo se renderiza el contenido.El primer paso para crear tu proyecto en HTML5, será crear un archivo index.html desde el éditor de código. Para ello, una vez instalado el mismo, ve a FILE->NEW FILE->(aquí debes nombrar a tu primer archivo "index.html" y lo guardas en una carpeta en donde tendrás tu proyecto).

Reglas semánticas de HTML

La semántica en HTML se refiere a usar las etiquetas correctas para describir el propósito y el significado del contenido. Esto mejora la accesibilidad, el SEO (optimización para motores de búsqueda) y la claridad del código. Algunas reglas importantes son:

  • Usa <header> para encabezados principales o de secciones.
  • Usa <main> para el contenido principal del documento, asegurándote de que sea único en cada página.
  • Organiza los artículos independientes o bloques de contenido usando <article>.
  • Usa <section> para agrupar contenido relacionado dentro de una página o artículo.
  • Para contenido adicional o complementario, como barras laterales, utiliza <aside>.
  • Usa <footer> para contenido al final de una página o sección, como derechos de autor o enlaces importantes.

Descripción de las etiquetas semánticas más importantes

  • <header>: Representa un encabezado para un documento o sección. Puede contener títulos, logotipos, menús de navegación o cualquier otro contenido introductorio. 
    <header>
            <h1>Título principal</h1>
            <nav>...</nav>
        </header>
  • <nav>: Define una sección de navegación con enlaces a otras partes del sitio o página. 
    <nav>
            <ul>
                <li><a href="#inicio">Inicio</a></li>
                <li><a href="#contacto">Contacto</a></li>
            </ul>
        </nav>
  • <main>: Indica el contenido principal de la página. Solo debe haber un <main> por documento. 
    <main>
            <article>...</article>
            <section>...</section>
        </main>
  • <article>: Representa contenido independiente, como un artículo de blog, un comentario o un elemento en un feed de noticias. 
    <article>
            <h2>Título del artículo</h2>
            <p>Contenido del artículo.</p>
        </article>
  • <section>: Agrupa contenido temáticamente relacionado dentro de una página o artículo. 
    <section>
            <h2>Subtítulo de la sección</h2>
            <p>Contenido relacionado.</p>
        </section>
  • <aside>: Contiene contenido adicional o complementario, como barras laterales, anuncios o widgets. 
    <aside>
            <p>Enlaces relacionados o información extra.</p>
        </aside>
  • <footer>: Define el pie de página de un documento o sección. Puede incluir información de contacto, derechos de autor, o enlaces adicionales. 
    <footer>
            <p>© 2025 Mi Sitio Web</p>
        </footer>

Estas etiquetas semánticas son clave para crear páginas web bien organizadas, accesibles y fáciles de entender tanto para los usuarios como para los motores de búsqueda. Siempre es una buena práctica usarlas correctamente para mantener estándares de calidad y accesibilidad.

Metadatos en HTML

Los metadatos en HTML son información sobre el contenido de la página web que no se muestra directamente, pero que es útil para los navegadores, motores de búsqueda, redes sociales y otras aplicaciones. Se definen dentro del elemento <head> y son esenciales para mejorar la accesibilidad y la visibilidad de una página en la web.

¿Qué son los metadatos?

Los metadatos son datos que describen otros datos. En el contexto de HTML, se refieren a información sobre el contenido de una página web, como su título, descripción, palabras clave, autor, y otros detalles que pueden ser utilizados por los navegadores y motores de búsqueda para mejorar la experiencia del usuario y optimizar el SEO (Search Engine Optimization).

Tipos de Metadatos en HTML

Existen varios tipos de metadatos que se pueden incluir dentro del <head> de una página HTML. A continuación, se describen los más comunes:

Título de la Página

El título de una página es un metadato importante que aparece en la barra de título del navegador y es utilizado por los motores de búsqueda. Se define dentro de la etiqueta <title>.

<title>Mi Página Web</title>

Descripción de la Página

La descripción proporciona un resumen breve del contenido de la página y es utilizada por los motores de búsqueda en sus resultados. Se define con la etiqueta <meta> con el atributo name="description".

<meta name="description" content="Este es un ejemplo de página web sobre metadatos en HTML">

Palabras Clave (Keywords)

Las palabras clave ayudan a los motores de búsqueda a entender de qué trata la página. Aunque su importancia ha disminuido, siguen siendo útiles en algunos casos. Se definen mediante <meta> con el atributo name="keywords".

<meta name="keywords" content="HTML, metadatos, SEO, página web">

Autor de la Página

Este metadato especifica quién es el autor del contenido de la página. Se define con la etiqueta <meta> y el atributo name="author".

<meta name="author" content="Juan Pérez">

Codificación de Caracteres (charset)

El metadato charset especifica la codificación de caracteres que se utiliza en la página. Es importante para asegurar que los caracteres especiales se muestren correctamente. Se define dentro de la etiqueta <meta>.

<meta charset="UTF-8">

¿Por qué son importantes los metadatos?

Los metadatos son esenciales para el funcionamiento de una página web por varias razones:

  • SEO (Optimización para motores de búsqueda): Los motores de búsqueda utilizan los metadatos para clasificar y mostrar las páginas web en sus resultados.
  • Redes Sociales: Las redes sociales utilizan metadatos para mostrar los enlaces de forma más atractiva, como el título, la descripción y la imagen de vista previa.
  • Accesibilidad: Los metadatos también mejoran la accesibilidad de la página, proporcionando información adicional que puede ser leída por herramientas de asistencia.

Metadatos Avanzados

Además de los metadatos básicos que hemos mencionado, existen otros tipos de metadatos más avanzados que pueden mejorar aún más la experiencia de usuario y la interacción con la página:

Open Graph

Open Graph es un conjunto de metadatos utilizados por redes sociales como Facebook y Twitter para mejorar la visualización de los enlaces compartidos. Estos metadatos incluyen información como el título, la descripción, la imagen y el tipo de contenido.

<meta property="og:title" content="Mi Página Web">

Twitter Cards

Las Twitter Cards son metadatos específicos para Twitter que permiten a los usuarios ver más contenido dentro de un tweet, como una imagen, un título y una descripción.

<meta name="twitter:title" content="Mi Página Web">

Viewport

El metadato viewport controla cómo se muestra la página en dispositivos móviles. Es fundamental para el diseño web responsivo y la experiencia móvil.

<meta name="viewport" content="width=device-width, initial-scale=1">

Conclusión

Los metadatos son una parte fundamental del desarrollo web moderno. No solo facilitan la indexación de la página por parte de los motores de búsqueda, sino que también mejoran la accesibilidad y la experiencia del usuario. Es recomendable incluir metadatos adecuados para aprovechar al máximo su potencial en términos de SEO, redes sociales y adaptabilidad en dispositivos móviles.

Etiquetas para texto

HTML permite estructurar texto usando etiquetas específicas que indican el propósito del contenido. Esto no solo ayuda a la presentación visual, sino que también mejora la semántica y accesibilidad del contenido web.

Estructura del texto con etiquetas más comunes

Las etiquetas principales para estructurar y dar formato al texto en HTML son:

  • <p>: Representa un párrafo. Es la forma más común de agrupar texto en bloques lógicos. 
    <p>Este es un párrafo de ejemplo.</p>
  • <h1>...<h6>: Representan encabezados jerárquicos, donde <h1> es el título principal y <h6> el menos importante.Sólo debe haber un h1 (título principal) y hasta h6, es decir, no existe h7 (puedes crear un elemento h7 personalizado con comportamiento similar a una etiqueta de encabezado usando Web Components y darle estilo y funcionalidad según tus necesidades, pero "esa es otra historia"👑⚔️. ) 
    <h1>Título Principal</h1>
        <h2>Subtítulo</h2>
  • <strong>: Indica texto de importancia o énfasis fuerte, generalmente se muestra en negrita. 
    <strong>Texto importante</strong>
  • <em>: Denota énfasis en el texto, generalmente se muestra en cursiva. 
    <em>Texto enfatizado</em>

Etiquetas para formatear texto

  • <blockquote>: Citas en bloque. Se utiliza para contenido citado de otra fuente. 
    <blockquote>
            "La tecnología avanza cada día más rápido."
        </blockquote>
  • <q>: Citas en línea. 
    <q>Cita breve dentro de un párrafo.</q>
  • <code>: Representa fragmentos de código. 
    <code>let x = 10;</code>
  • <pre>: Conserva el formato del texto tal como se escribe, incluyendo espacios y saltos de línea. 
    <pre>
            Código con formato exacto.
            Línea 1
            Línea 2
        </pre>
  • <mark>: Destaca texto como si estuviera resaltado. 
    <mark>Texto destacado</mark>
  • <del>: Representa texto eliminado o tachado. 
    <del>Texto eliminado</del>
  • <ins>: Representa texto insertado. 
    <ins>Texto agregado</ins>
  • <sup> y <sub>: Superíndices y subíndices, respectivamente. 
    E=mc<sup>2</sup> // H<sub>2</sub>O

Etiquetas HTML menos comunes pero útiles

  • <abbr>: Define abreviaturas. 
    <abbr title="HyperText Markup Language">HTML</abbr>
  • <time>: Representa un momento específico en el tiempo o una duración. 
    <time datetime="2025-01-13">13 de enero de 2025</time>
  • <address>: Representa información de contacto. 
    <address>
            Autor: Juan Pérez <br>
            Email: juan@ejemplo.com
        </address>

Ejemplo de etiqueta en desuso

HTML también tiene etiquetas en desuso, que ya no se recomiendan. Por ejemplo:

  • <font>: Se usaba para cambiar el tipo, tamaño o color de la fuente, pero ahora está en desuso porque el formato debe manejarse con CSS. 
    <font color="red" size="4">Texto en desuso</font>

    Forma moderna: Usa CSS:

    <span style="color:red; font-size:16px;">Texto recomendado</span>

Importancia de usar etiquetas modernas

Usar etiquetas modernas y semánticas ayuda a garantizar que los navegadores interpreten correctamente el contenido. También mejora la accesibilidad y la optimización para motores de búsqueda (SEO), asegurando que el contenido sea claro para humanos y máquinas.

Otras etiquetas

Vamos a complementar la explicación con las etiquetas que faltaron, como
, , , y otras que son importantes en HTML.

  • <br>: Inserta un salto de línea. Es una etiqueta vacía (sin cierre). 
    <p>Primera línea<br>Segunda línea</p>

    Nota: Úsala con moderación, para separar contenido es mejor usar etiquetas como <p>.

  • <span>: Contenedor en línea usado para aplicar estilos o comportamientos específicos dentro del texto. 
    <p>El precio es <span style="color: red;">$50</span>.</p>
  • <i>: Representa texto que debe destacarse visualmente (generalmente en cursiva). 
    <p>El término <i>HTML</i> es importante.</p>

    Nota: Para énfasis semántico usa <em>.

  • <small>: Representa texto en un tamaño más pequeño, útil para notas o aclaraciones. 
    <p>© 2025 <small>Todos los derechos reservados.</small></p>
  • <mark>: Resalta texto como si estuviera marcado. 
    <p>Palabra clave: <mark>HTML</mark>.</p>
  • <abbr>: Define una abreviatura o acrónimo. 
    <p>La organización <abbr title="World Wide Web Consortium">W3C</abbr> establece estándares web.</p>
  • <cite>: Representa el título de una obra.

  • <hr>: Inserta una línea horizontal para separar secciones. 
    <p>Primera sección</p><hr><p>Segunda sección</p>
  • <font>: Se usaba para cambiar el tipo, tamaño o color de la fuente, pero ahora está en desuso porque el formato debe manejarse con CSS. 
    <font color="red" size="4">Texto en desuso</font>

¿Qué es Schema.org y por qué usar microdatos?

Usar microdatos con Schema.org en HTML es una práctica común y valiosa, especialmente en un sitio de comercio electrónico, también conocide como E-Commerce. Estos microdatos ayudan a estructurar mejor la información de tu sitio web, proporcionando datos específicos que los motores de búsqueda pueden interpretar fácilmente para mostrar resultados enriquecidos (rich snippets) en las páginas de resultados (SERPs).

¿Qué es Schema.org?

Schema.org es una iniciativa conjunta de Google, Microsoft, Yahoo y Yandex para crear un vocabulario estandarizado que describa datos en sitios web. Este vocabulario es interpretado por motores de búsqueda para entender mejor el contenido y mejorar la presentación de los resultados. Al utilizar este vocabulario, los motores de búsqueda pueden mostrar información más detallada y precisa sobre los elementos de tu página, como productos, reseñas o eventos, directamente en los resultados de búsqueda.

¿Por qué usar microdatos en un e-commerce?

En un sitio de comercio electrónico, puedes usar microdatos para:

  • Mostrar productos en los resultados de búsqueda: Incluye detalles como nombre, precio, disponibilidad y calificación.
  • Aumentar la visibilidad y el CTR: Destacar información clave directamente en los resultados puede atraer más clics.
  • Mejorar la accesibilidad y la experiencia del usuario: Facilita la comprensión de los datos para motores de búsqueda y asistentes virtuales, mejorando la interacción del usuario con el sitio.

Ventajas prácticas de usar microdatos en un e-commerce

  • Resultados enriquecidos: Los productos pueden aparecer con calificaciones, precios y disponibilidad en los resultados de búsqueda, lo que mejora la visibilidad y la tasa de clics.
  • Compatibilidad con asistentes de voz: Los asistentes como Google Assistant pueden utilizar estos datos para responder preguntas de usuarios de manera más precisa.
  • Mayor confianza: Los clientes tienen más información antes de visitar tu sitio, lo que puede aumentar la probabilidad de conversión.

Otros esquemas útiles para e-commerce

  • Oferta especial: Usa https://schema.org/Offer para describir ofertas especiales o descuentos.
  • Reseñas individuales: Usa https://schema.org/Review para mostrar opiniones detalladas de productos o servicios.
  • Breadcrumbs: Ayuda a mejorar la navegación de tu sitio, usando https://schema.org/BreadcrumbList.
  • Eventos de ventas o descuentos: Usa https://schema.org/SaleEvent para describir eventos de ventas o promociones especiales.

Enlaces en HTML

En HTML, los enlaces se crean utilizando la etiqueta <a> y las imágenes con la etiqueta <img>. Ambas son esenciales para la navegación y presentación de contenido visual en páginas web.

Enlaces (<a>)

La etiqueta <a>, conocida como anchor (ancla), se usa para crear hipervínculos, es decir, enlaces a otras páginas, documentos, o recursos dentro de un sitio web o fuera de él. Un enlace se define con el atributo href, que especifica la dirección (URL) a la que se dirige el enlace.

Ejemplo básico de un enlace:

<a href="https://www.example.com">Visita nuestro sitio</a>

En este caso, el texto "Visita nuestro sitio" se convierte en un enlace que llevará al usuario a www.example.com.

Atributos comunes de los enlaces:

  • href: Define la URL de destino del enlace.
  • target: Especifica cómo se abrirá el enlace. El valor más común es _blank, que abre el enlace en una nueva ventana o pestaña del navegador. Por ejemplo: <a href="https://www.example.com" target="_blank">Enlace</a>.
  • title: Proporciona información adicional sobre el enlace cuando el usuario pasa el cursor sobre él. Ejemplo: <a href="https://www.example.com" title="Ir al sitio de ejemplo">Enlace con título</a>
  • rel: Define la relación entre la página actual y la página a la que enlaza. Por ejemplo, rel="noopener noreferrer" se usa para mejorar la seguridad cuando se abren enlaces en nuevas pestañas.

Imágenes en HTML(<img>)

La etiqueta <img> se utiliza para insertar imágenes en una página web. A diferencia de otras etiquetas, <img> es un elemento vacío, lo que significa que no tiene una etiqueta de cierre. Los atributos principales de esta etiqueta son src y alt.

Ejemplo básico de una imagen:

<img src="imagen.jpg" alt="Descripción de la imagen">

En este caso, la imagen "imagen.jpg" se mostrará en la página. Si la imagen no se puede cargar, se mostrará el texto alternativo ("Descripción de la imagen").

Ejemplo imágen

Atributos comunes de las imágenes:

  • src: Define la ruta o URL de la imagen. Por ejemplo: <img src="logo.png">.
  • alt: Proporciona un texto alternativo que describe la imagen. Este texto se muestra si la imagen no se puede cargar o si el usuario tiene una discapacidad visual y usa un lector de pantalla. Ejemplo: <img src="logo.png" alt="Logotipo de la empresa">.
  • width y height: Especifican el tamaño de la imagen en píxeles. Aunque se recomienda usar CSS para el control de diseño, puedes especificar un tamaño en el HTML directamente. Ejemplo: <img src="foto.jpg" width="300" height="200">.
  • title: Muestra un texto cuando el usuario pasa el cursor sobre la imagen. Ejemplo: <img src="logo.png" alt="Logotipo" title="Haz clic para ver más">.

Uso de srcset y la Etiqueta

1. Atributo srcset en la etiqueta <img>

El atributo srcset en la etiqueta <img> se utiliza para especificar diferentes versiones de una imagen, permitiendo que el navegador seleccione la más adecuada según el tamaño y la resolución de la pantalla del usuario. Esto es especialmente útil para crear páginas web responsivas y optimizar el rendimiento.

¿Cómo funciona el atributo srcset?

El atributo srcset permite proporcionar una lista de imágenes alternativas, junto con una descripción de sus características (como el ancho en píxeles o el factor de densidad de la pantalla). El navegador seleccionará automáticamente la imagen más adecuada.


                <img src="imagen-predeterminada.jpg" 
                srcset="imagen-pequena.jpg 480w, 
                    imagen-mediana.jpg 800w, 
                    imagen-grande.jpg 1200w" 
                sizes="(max-width: 600px) 480px, 
                    (max-width: 1200px) 800px, 
                    1200px" 
                alt="Descripción de la imagen">

Explicación de los atributos:

  • src: Proporciona la URL de la imagen predeterminada (usada por navegadores que no soportan srcset o si no se cumplen las condiciones de las imágenes alternativas).
  • srcset: Define una lista de imágenes alternativas, cada una seguida de un descriptor:
    • 480w, 800w, etc.: Especifica el ancho en píxeles de la imagen.
    • 1x, 2x, etc. (opcional): Indica el factor de densidad de la imagen para pantallas con diferentes resoluciones (como pantallas Retina).
  • sizes: Indica el ancho del espacio en el diseño en el que se mostrará la imagen (en unidades CSS) dependiendo de las condiciones del dispositivo, como el ancho de la pantalla:
    • (max-width: 600px) 480px: Si la pantalla tiene menos de 600px de ancho, usa una imagen de 480px.
    • (max-width: 1200px) 800px: Si la pantalla tiene entre 600px y 1200px, usa una imagen de 800px.
    • 1200px: En pantallas más grandes, usa una imagen de 1200px.

Nota: El atributo sizes puede ser reemplazado por CSS, lo que simplifica el manejo de los tamaños en la mayoría de los casos.

2. La etiqueta <figure> en HTML

La etiqueta <figure> se utiliza para agrupar contenido independiente, generalmente relacionado con componentes visuales como imágenes, diagramas, gráficos, fragmentos de código, etc. Se usa comúnmente junto con la etiqueta <figcaption> para proporcionar un título o descripción del contenido.


        <figure>
            <img src="imagen.jpg" alt="Descripción de la imagen">
            <figcaption>Descripción de la imagen o contenido relacionado.</figcaption>
        </figure>

Explicación de los elementos:

  • <figure>: Agrupa el contenido visual y su descripción. Es semánticamente significativo y facilita la comprensión del contenido por parte de los motores de búsqueda y lectores de pantalla.
  • <img>: Representa el contenido visual, como una imagen o un gráfico.
  • <figcaption>: Proporciona una leyenda para el contenido dentro de <figure>. Es opcional, pero útil para añadir contexto o descripción.

Ventajas de usar <figure> y <figcaption>:

  • Accesibilidad: Mejora la experiencia de usuarios que utilizan lectores de pantalla, ya que el contenido y su descripción están semánticamente relacionados.
  • SEO: Los motores de búsqueda pueden entender mejor el propósito del contenido visual y su relación con el texto.
  • Flexibilidad: Es útil para cualquier contenido visual o referencial, no solo imágenes.
  • Separación lógica: Facilita el diseño y la estructura del documento HTML.

Estilos con CSS: Puedes personalizar <figure> y <figcaption> con CSS para mejorar su presentación, como ajustar márgenes, fondos y tipografía. Incluso puedes envolver fragmentos de código en <figure> para destacarlos dentro del contenido.

Rutas Absolutas y Relativas en HTML

En HTML, las rutas absolutas y relativas se utilizan para enlazar a recursos como páginas, imágenes, hojas de estilo o scripts. Aquí está la diferencia:

Ruta Absoluta

Es una dirección completa que incluye el protocolo (como http:// o https://) y el dominio. Esta ruta apunta a un recurso externo o específico sin importar la ubicación del archivo actual.

Ejemplo de ruta absoluta:

<a href="https://www.ejemplo.com/pagina.html">Página Absoluta</a>
<img src="https://www.ejemplo.com/imagenes/logo.png" alt="Logo">

Características:

  • Funciona desde cualquier lugar.
  • Utilizada generalmente para recursos externos.
  • Incluye toda la dirección del recurso.

Ruta Relativa

Es una dirección parcial basada en la ubicación actual del archivo. Se utiliza para enlazar recursos dentro del mismo sitio o proyecto.

Tipos de Rutas Relativas:

1. Ruta en el mismo directorio

Cuando el archivo está en el mismo directorio que el recurso:

<a href="pagina.html">Página Relativa</a>
<img src="logo.png" alt="Logo">
2. Ruta en un subdirectorio

Cuando el recurso está en un directorio dentro del actual:

<a href="subcarpeta/pagina.html">Subdirectorio</a>
<img src="imagenes/logo.png" alt="Logo">
3. Ruta en un directorio superior

Cuando se necesita acceder al directorio padre usando ../:

<a href="../pagina.html">Directorio Superior</a>
<img src="../imagenes/logo.png" alt="Logo">
4. Ruta desde la raíz del proyecto

Usando /, se hace referencia al directorio raíz del servidor o del proyecto:

<a href="/inicio.html">Inicio</a>
<img src="/imagenes/logo.png" alt="Logo">

Características:

  • Depende de la ubicación del archivo actual.
  • Es útil para organizar proyectos y facilitar cambios de dominio o rutas.
  • Más comúnmente utilizada en proyectos internos o locales.

Enlaces con Atributo download

Puedo usar un enlace con el atributo download para descargar archivos directamente desde un enlace, como en este ejemplo:

<a href="pexels.io/manzanas" download="manzanas.jpg">Descargar manzanas</a>

El atributo download le indica al navegador que, en lugar de abrir el archivo, debe descargarlo.

Marcadores en HTML

Los marcadores se utilizan para navegar a diferentes secciones de una misma página. Para hacerlo, se utiliza el atributo id en un elemento de destino y un enlace con href apuntando al id:

Definir un marcador con id:

<div id="seccion1">Contenido de la sección 1</div>

Enlazar a la sección usando <a href="#nombredelID">:

<a href="#seccion1">Ir a la Sección 1</a>

Esto hará que el navegador se desplace automáticamente a la sección con el id="seccion1" cuando se haga clic en el enlace.

Características:

  • Permite la navegación interna dentro de la misma página.
  • Facilita la organización de contenido largo o complejo en una página web.
  • Es común en tablas de contenido o menús de navegación.

Listas en HTML

En HTML, las listas se utilizan para organizar elementos en una secuencia o en grupos. Existen dos tipos principales de listas: listas ordenadas y listas desordenadas. Cada tipo tiene su propio formato y propósito, dependiendo de cómo se quiere presentar la información.

Lista Ordenada (<ol>)

Una lista ordenada se usa cuando los elementos tienen un orden específico. En este tipo de lista, cada ítem está numerado automáticamente por el navegador, lo que indica que la secuencia es importante. Se usa la etiqueta <ol> para crear una lista ordenada, y cada ítem se define dentro de la lista usando la etiqueta <li>.

Ejemplo de una lista ordenada:


        <ol>
            <li>Primer paso</li>
            <li>Segundo paso</li>
            <li>Tercer paso</li>
        </ol>

Esto generará una lista numerada en la página:

  1. Primer paso
  2. Segundo paso
  3. Tercer paso

Atributos de las listas ordenadas

En las listas ordenadas, puedes usar algunos atributos para personalizar la numeración:

  • start: Define el número con el que debe comenzar la numeración.
  • type: Cambia el tipo de marcador (puede ser 1 para números, A para letras mayúsculas, a para letras minúsculas, etc.).
  • reversed: Invierte el orden de la numeración, comenzando desde el número más alto.

Ejemplo:


        <ol start="5" reversed type="A">
            <li>Quinto paso</li>
            <li>Cuarto paso</li>
            <li>Tercer paso</li>
        </ol>

Esto generará una lista ordenada que comienza en la letra "A" y va en orden inverso:

  1. Quinto paso
  2. Cuarto paso
  3. Tercer paso

Lista Desordenada (<ul>)

Una lista desordenada se usa cuando el orden de los elementos no es relevante. En este tipo de lista, cada ítem está precedido por un marcador (generalmente un círculo, pero esto depende de la configuración del navegador). Se usa la etiqueta <ul> para crear una lista desordenada, y cada ítem se define con la etiqueta <li>.

Ejemplo de una lista desordenada:


        <ul>
            <li>Manzanas</li>
            <li>Bananas</li>
            <li>Cerezas</li>
        </ul>

Esto generará una lista con viñetas:

  • Manzanas
  • Bananas
  • Cerezas

Atributos de las listas desordenadas

En las listas desordenadas, el atributo principal es el list-style-type, que define el tipo de marcador usado en la lista. Se puede usar en el elemento <ul> o en el CSS:

  • disc: El marcador por defecto, representado por un círculo lleno.
  • circle: Un marcador representado por un círculo vacío.
  • square: Un marcador representado por un cuadrado lleno.

Ejemplo con list-style-type:


        <ul style="list-style-type: square">
            <li>Elemento 1</li>
            <li>Elemento 2</li>
            <li>Elemento 3</li>
        </ul>

Esto generará una lista con marcadores en forma de cuadrados:

  • Elemento 1
  • Elemento 2
  • Elemento 3

Listas Anidadas

Las listas también pueden ser anidadas, es decir, puedes colocar una lista dentro de otra. Esto es útil cuando tienes subcategorías o elementos dentro de un elemento principal. En una lista ordenada o desordenada, puedes agregar otro <ul> o <ol> dentro de un <li>.

Ejemplo de lista anidada:


        <ul>
            <li>Frutas
                <ul>
                    <li>Manzanas</li>
                    <li>Bananas</li>
                </ul>
            </li>
            <li>Verduras
                <ul>
                    <li>Espinacas</li>
                    <li>Zanahorias</li>
                </ul>
            </li>
        </ul>

Esto generará una lista con elementos anidados:

  • Frutas
    • Manzanas
    • Bananas
  • Verduras
    • Espinacas
    • Zanahorias

Listas de Definición (<dl>)

Además de las listas ordenadas y desordenadas, HTML también ofrece un tipo de lista llamado lista de definición. Esta lista se utiliza para mostrar definiciones de términos. Está compuesta por los siguientes elementos:

  • <dl>: Define la lista de definición.
  • <dt>: Define el término a ser descrito.
  • <dd>: Define la descripción o definición del término.

Ejemplo de una lista de definición:


        <dl>
            <dt>HTML</dt>
            <dd>Es el lenguaje de marcado utilizado para crear páginas web.</dd>
            <dt>CSS</dt>
            <dd>Es el lenguaje utilizado para diseñar y dar estilo a las páginas web.</dd>
        </dl>

Esto generará una lista de definición:

HTML
Es el lenguaje de marcado utilizado para crear páginas web.
CSS
Es el lenguaje utilizado para diseñar y dar estilo a las páginas web.

Formularios

Los formularios en HTML son elementos fundamentales para la interacción con los usuarios. Se utilizan para recopilar información y enviarla a un servidor para su procesamiento. Un formulario se estructura principalmente con la etiqueta <form> y varios elementos interactivos como <input>, <textarea>, <select>, y <button>.

1. Estructura de un formulario

La estructura básica de un formulario se crea con la etiqueta <form>. Esta etiqueta agrupa los controles del formulario, como campos de texto, botones, menús desplegables, etc. A continuación se muestra un ejemplo básico:

<form action="procesar.php" method="post">
        <label for="nombre">Nombre:</label>
        <input type="text" id="nombre" name="nombre" required>
    
        <label for="email">Correo Electrónico:</label>
        <input type="email" id="email" name="email" required>
    
        <label for="mensaje">Mensaje:</label>
        <textarea id="mensaje" name="mensaje"></textarea>
    
        <button type="submit">Enviar</button>
    </form>

2. Elementos clave en formularios

Existen varios tipos de elementos dentro de un formulario que permiten interactuar con el usuario. Los más comunes son:

  • <input>: Se usa para campos de texto, contraseñas, números, etc.
  • <textarea>: Se utiliza para entradas de texto más largas, como comentarios o mensajes.
  • <select>: Permite crear listas desplegables.
  • <button>: Se usa para enviar el formulario o realizar acciones.

A continuación, te mostramos ejemplos de estos elementos:

<input type="text" name="usuario" placeholder="Ingrese su usuario">
<textarea name="comentarios" rows="4" cols="50"></textarea>
<select name="opciones">
        <option value="opcion1">Opción 1</option>
        <option value="opcion2">Opción 2</option>
        <option value="opcion3">Opción 3</option>
    </select>

3. Validación de formularios

HTML5 proporciona varias herramientas para la validación de formularios antes de que se envíen al servidor. Algunos de los atributos de validación más comunes son:

  • required: Hace que un campo sea obligatorio.
  • pattern: Permite definir una expresión regular para validar el formato del dato.
  • min y max: Establecen los valores mínimo y máximo para ciertos campos (como números o fechas).

Ejemplo con validación básica:

<form action="procesar.php" method="post">
        <label for="nombre">Nombre:</label>
        <input type="text" id="nombre" name="nombre" required>
    
        <label for="email">Correo Electrónico:</label>
        <input type="email" id="email" name="email" required>
    
        <label for="contraseña">Contraseña:</label>
        <input type="password" id="contraseña" name="contraseña" required>
    
        <button type="submit">Enviar</button>
    </form>

4. Elementos adicionales: <fieldset>, <legend>, y <label>

Para mejorar la accesibilidad y organización de los formularios, puedes usar los elementos <fieldset> y <legend>. El elemento <fieldset> agrupa elementos relacionados, mientras que <legend> proporciona una descripción de esos elementos.

<form>
        <fieldset>
            <legend>Información Personal</legend>
            <label for="name">Nombre:</label>
            <input type="text" id="name" name="name">
            <br>
            <label for="age">Edad:</label>
            <input type="number" id="age" name="age">
        </fieldset>
    </form>

Los elementos <label> son importantes para asociar texto con campos de formulario. Mejora la accesibilidad y permite que los usuarios hagan clic en el texto para seleccionar el campo correspondiente:

<label for="username">Usuario:</label>
    <input type="text" id="username" name="username">

5. Menú desplegable y selección múltiple

El elemento <select> se utiliza para crear menús desplegables, donde el usuario puede seleccionar una opción. Si necesitas permitir múltiples selecciones, puedes agregar el atributo multiple.

<label for="colors">Elige un color:</label>
    <select id="colors" name="colors">
        <option value="red">Rojo</option>
        <option value="green">Verde</option>
        <option value="blue">Azul</option>
    </select>

6. Área de texto

El elemento <textarea> permite que el usuario ingrese texto largo, como comentarios o descripciones. Se puede personalizar con atributos como rows y cols para ajustar el tamaño, o maxlength para limitar la cantidad de caracteres ingresados.

<textarea name="comentarios" rows="4" cols="50"></textarea>

7. Estilización de formularios

Los formularios y sus elementos se pueden estilizar utilizando CSS para mejorar su apariencia. Puedes cambiar el tamaño, color, borde y más, usando reglas CSS.

textarea {
        width: 100%;
        height: 150px;
        border: 1px solid #ccc;
        padding: 10px;
        font-size: 16px;
    }

1. Formulario de entrada de texto

Este es un campo de texto básico. Se utiliza cuando se espera que el usuario ingrese un texto corto, como un nombre o un apellido.

2. Formulario de correo electrónico

Este campo es para ingresar una dirección de correo electrónico. El navegador validará automáticamente el formato del correo antes de enviarlo.

3. Formulario de contraseña

El campo de contraseña es utilizado para ingresar una clave secreta. El texto ingresado se oculta por razones de seguridad.

4. Formulario de selección de color

Este formulario permite que el usuario seleccione un color usando un selector de color. El valor por defecto es el rojo (#ff0000).

5. Formulario con área de texto

El campo <textarea> permite que el usuario ingrese texto largo, como comentarios, descripciones o mensajes.

6. Formulario de selección múltiple

Este formulario permite que el usuario seleccione varias opciones de una lista desplegable. El atributo multiple permite que se elijan más de una opción.

7. Formulario de radio buttons

¿Prefieres el café o el té?

Los botones de opción o radio permiten que el usuario seleccione solo una opción de un conjunto de opciones disponibles.

8. Formulario de casillas de verificación (Checkbox)

La casilla de verificación se usa cuando se quiere que el usuario acepte o confirme algo, como los términos y condiciones. El atributo required asegura que el usuario debe marcarla para continuar.

9. Formulario con un botón de envío

El botón de envío, creado con la etiqueta <button>, envía el formulario al servidor para ser procesado.

Tablas en HTML

Las tablas en HTML se crean utilizando la etiqueta <table>, junto con otras etiquetas como <tr>, <th> y <td> para definir las filas, los encabezados de columna y las celdas de la tabla, respectivamente.

1. Estructura Básica de una Tabla

Una tabla básica en HTML está compuesta por la etiqueta <table>, que contiene las etiquetas <tr> (fila de la tabla), <th> (encabezado de columna) y <td> (celda de datos).


    <table>
        <tr>
            <th>Encabezado 1</th>
            <th>Encabezado 2</th>
        </tr>
        <tr>
            <td>Dato 1</td>
            <td>Dato 2</td>
        </tr>
    </table>

El código anterior crea una tabla con dos columnas (Encabezado 1 y Encabezado 2) y una fila con datos (Dato 1 y Dato 2).

2. Explicación de las Etiquetas de la Tabla

  • <table>: Define el contenedor de la tabla.
  • <tr>: Representa una fila dentro de la tabla.
  • <th>: Representa una celda de encabezado, generalmente utilizada para los títulos de las columnas. Por defecto, el texto dentro de <th> aparece en negrita y centrado.
  • <td>: Representa una celda de datos dentro de una fila. Contiene el contenido de la tabla.

3. Ejemplo de una Tabla Completa

A continuación, se muestra un ejemplo de una tabla con varias filas y columnas, incluyendo tanto datos como encabezados:


    <table border="1">
        <tr>
            <th>Nombre</th>
            <th>Edad</th>
            <th>Profesión</th>
        </tr>
        <tr>
            <td>Juan</td>
            <td>30</td>
            <td>Ingeniero</td>
        </tr>
        <tr>
            <td>María</td>
            <td>25</td>
            <td>Diseñadora</td>
        </tr>
        <tr>
            <td>Carlos</td>
            <td>28</td>
            <td>Médico</td>
        </tr>
    </table>

En este ejemplo, la tabla tiene tres columnas: Nombre, Edad y Profesión, y tres filas de datos.

4. Personalización de Tablas con Atributos

Existen varios atributos que puedes usar para personalizar la apariencia de las tablas, como border, cellspacing, cellpadding y width.

  • border: Define el grosor del borde de la tabla.
  • cellspacing: Define el espacio entre las celdas de la tabla.
  • cellpadding: Define el espacio dentro de cada celda (margen entre el borde de la celda y el contenido).
  • width: Define el ancho de la tabla o las celdas.

A continuación, te mostramos un ejemplo de cómo agregar estos atributos:


    <table border="1" cellspacing="5" cellpadding="10" width="100%">
        <tr>
            <th>Producto</th>
            <th>Precio</th>
        </tr>
        <tr>
            <td>Café</td>
            <td>$3.00</td>
        </tr>
        <tr>
            <td>Té</td>
            <td>$2.50</td>
        </tr>
    </table>

En este ejemplo, hemos añadido un borde, espacio entre celdas, relleno dentro de las celdas y un ancho del 100% de la tabla.

Contenido Multimedia

HTML soporta audio y video con las etiquetas <audio> y <video>.

Etiquetas <audio> y <video>

Las etiquetas <audio> y <video> en HTML se utilizan para incorporar archivos de audio y video en una página web, ofreciendo controles para reproducir, pausar y ajustar el volumen, entre otras funcionalidades. Ambas etiquetas forman parte de HTML5 y son altamente versátiles.

Etiqueta <audio>

Se utiliza para insertar contenido de solo audio, como música o efectos de sonido.

Ejemplo básico
<audio controls>
        <source src="audio.mp3" type="audio/mpeg">
        <source src="audio.ogg" type="audio/ogg">
        Tu navegador no soporta la reproducción de audio.
    </audio>
Atributos comunes de <audio>
  • controls: Muestra controles de reproducción como play, pause y volumen.
  • autoplay: El audio se reproduce automáticamente al cargar la página.
  • loop: El audio se reproduce en bucle continuo.
  • muted: Inicia el audio con el sonido silenciado.
  • preload: Sugerencia sobre cómo cargar el audio:
    • auto: Carga completo el archivo al cargar la página.
    • metadata: Carga solo la duración y otros metadatos.
    • none: No carga el archivo hasta que el usuario lo reproduzca.

Etiqueta <video>

Se utiliza para incrustar contenido de video, y puede incluir subtítulos y controles personalizados.

Ejemplo básico
<video controls width="640" height="360">
        <source src="video.mp4" type="video/mp4">
        <source src="video.ogg" type="video/ogg">
        Tu navegador no soporta la reproducción de video.
    </video>
Atributos comunes de <video>
  • controls: Muestra controles como play, pause, barra de progreso y volumen.
  • autoplay: El video se reproduce automáticamente al cargar la página.
  • loop: El video se reproduce en bucle continuo.
  • muted: Inicia el video con el sonido silenciado.
  • poster: Define una imagen que se muestra antes de que se reproduzca el video.
  • preload: Sugerencia para la carga del video (similar a <audio>).

Subtítulos con <track>

Se puede añadir subtítulos, descripciones y otros metadatos a los videos mediante la etiqueta <track>.

Ejemplo con subtítulos
<video controls width="640" height="360">
        <source src="video.mp4" type="video/mp4">
        <track src="subtitulos.vtt" kind="subtitles" srclang="es" label="Español">
        Tu navegador no soporta la reproducción de video.
    </video>
Explicación de <track>:
  • src: Ruta del archivo de subtítulos (formato .vtt).
  • kind: Tipo de pista (e.g., subtitles, descriptions, captions).
  • srclang: Idioma de los subtítulos (e.g., es para español, en para inglés).
  • label: Texto que aparece para seleccionar los subtítulos.

Accesibilidad en HTML

HTML soporta atributos como alt y aria-* para hacer sitios más accesibles. La accesibilidad web se refiere a la práctica de diseñar y desarrollar sitios web de manera que sean accesibles para todas las personas, incluidas aquellas con discapacidades. A través de etiquetas y atributos semánticos, podemos hacer que nuestro contenido sea más comprensible y usable para todos los usuarios, incluidos aquellos que usan tecnologías de asistencia como lectores de pantalla.

Atributo alt en imágenes

El atributo alt se usa en la etiqueta <img> para proporcionar una descripción alternativa de una imagen. Este atributo es crucial para usuarios con discapacidades visuales, ya que los lectores de pantalla leen el contenido de este atributo en lugar de mostrar la imagen.

Atributos aria-* para mejorar la accesibilidad

Los atributos aria-* (Accessible Rich Internet Applications) son un conjunto de atributos que se pueden agregar a los elementos HTML para mejorar la accesibilidad, especialmente cuando se trabaja con contenido dinámico o interactivo.

Ejemplo de aria-label

El atributo aria-label proporciona una etiqueta accesible para los elementos interactivos. Esto es útil cuando el elemento no tiene una etiqueta visible o su etiqueta visual no describe adecuadamente su propósito.

Ejemplo de aria-hidden

El atributo aria-hidden se usa para ocultar elementos a los lectores de pantalla. Esto es útil cuando tienes elementos decorativos que no deberían ser leídos por los lectores de pantalla.

Ejemplo de aria-live

El atributo aria-live se utiliza para indicar que un elemento se actualizará dinámicamente, lo que significa que el lector de pantalla debe anunciar los cambios a medida que ocurren.

Este mensaje se actualizará sin interrumpir el flujo de lectura.

Ejemplo de tabindex para navegación por teclado

El atributo tabindex permite controlar el orden de enfoque de los elementos en una página cuando el usuario navega con el teclado (con la tecla Tab). Un valor positivo indica el orden de tabulación. Un valor de "0" permite que el elemento siga el flujo normal de tabulación.

El DOM (Document Object Model) es una representación estructurada de un documento HTML o XML como un árbol de nodos. Es la interfaz que permite a los lenguajes de programación (como JavaScript) acceder y manipular los elementos de la página.

Características del DOM

  • Representa la estructura jerárquica de un documento.
  • Cada etiqueta HTML es un nodo en el árbol DOM.

BOM (Browser Object Model)

El BOM es la representación de las funciones y objetos proporcionados por el navegador web para interactuar con el entorno fuera del documento HTML.

Elementos del BOM

  1. window: Es el objeto principal que representa la ventana del navegador.
  2. navigator: Proporciona información sobre el navegador y el sistema operativo.
  3. screen: Información sobre la resolución y dimensiones de la pantalla.
  4. location: Permite acceder o manipular la URL actual.
  5. history: Maneja el historial de navegación.

Ejemplos del BOM

            // Mostrar la URL actual
            console.log(window.location.href);
    
            // Redirigir a otra página
            window.location.href = "https://www.google.com";
    
            // Mostrar el historial del navegador
            console.log(window.history.length);

Web Storage

El Web Storage permite almacenar datos en el navegador de manera local. Ofrece dos tipos de almacenamiento:

  • localStorage: Los datos persisten incluso después de cerrar el navegador.
  • sessionStorage: Los datos se eliminan al cerrar la pestaña o ventana.

WebSockets

Los WebSockets permiten establecer una comunicación bidireccional en tiempo real entre el navegador y un servidor. Son útiles para aplicaciones que requieren actualizaciones instantáneas, como chats, juegos en línea, o paneles de monitoreo.

Cómo funcionan los WebSockets

  • Se establece una conexión persistente entre el cliente y el servidor.
  • Tanto el cliente como el servidor pueden enviar y recibir mensajes en cualquier momento.

Ejemplo básico de WebSocket

            const socket = new WebSocket("wss://echo.websocket.events");
    
            // Abrir conexión
            socket.onopen = () => {
                console.log("Conexión establecida");
                socket.send("Hola, servidor");
            };
    
            // Escuchar mensajes
            socket.onmessage = (event) => {
                console.log("Mensaje recibido:", event.data);
            };
    
            // Manejar errores
            socket.onerror = (error) => {
                console.error("Error en la conexión:", error);
            };
    
            // Cerrar conexión
            socket.onclose = () => {
                console.log("Conexión cerrada");
            };

Web Components

Los Web Components son una tecnología que permite crear elementos personalizados reutilizables con su propio comportamiento, estilo y funcionalidad. Son parte del estándar de HTML5.

Características principales

  1. Custom Elements: Definir nuevos elementos HTML personalizados.
  2. Shadow DOM: Encapsula el estilo y la funcionalidad de un componente para evitar conflictos con el resto de la página.
  3. HTML Templates: Permiten definir estructuras HTML reutilizables.
